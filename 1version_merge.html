Concept: Hidden Steps

Computers do more than the body statement. Trace every step.


for (int i = 0; i < 4; i++) {
    System.out.println("Hi");
}

Check i < 4: 5 times

(0,1,2,3 true + 4 false)

Update i++: 4 times

Body: 4 times

Rule: condition = iterations + 1

Fast Formula (AP Exam)

Count = End - Start + 1

Example 1

for(i = 0; i <= 10; i++)

10 - 0 + 1 = 11

Example 2

for(i = 5; i < 10; i++)

Last is 9 → 9 - 5 + 1 = 5

The Simple Loop (m × n grids)


for (int i = 0; i < 5; i++) {
    System.out.println("AP CSA");
}

Init: 1 time

Test: 6 times (5 true + 1 false)

Update: 5 times

Body: 5 times

Nested Loops · From Rows × Cols to N²


for (int r = 0; r < n; r++) {      // outer
  for (int c = 0; c < n; c++) {    // inner
    System.out.print("*");         // body
  }
  System.out.println();
}

Outer: $n$ times · Inner: $n$ times for each outer

Total = $n \times n = n^2$

If $n$ doubles → work x4. Structure beats hardware.

Reflection

Why it matters

  • Efficiency: Saves energy & cost.
  • User Experience: Avoids the endless spinner.
  • Ethics: Bad code can become a DoS.
Exit Ticket: Put an N-loop inside another N-loop. Complexity? Linear or Quadratic?

Summary

  • Execution Count: Count steps (incl. tests), not wall-clock.
  • Single Loop: ~n times (condition = n+1).
  • Nested Loop: ~n × n times (quadratic).
  • IB Mindset: justify with math, question structure.
