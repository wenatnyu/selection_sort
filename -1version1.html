"How many times does the computer have to work?"

[cite_start]Citing document[cite: 5]. Our goal isn't to measure seconds or milliseconds. It's to count "steps".

Why do we count?

Scenario A


// n is 1,000,000
for (int i = 0; i < n; i++) {
    System.out.println("Hi");
}

Runs 1 million times.

Scenario B


// n is 1,000,000
for (int i = 0; i < n; i++) {
    for (int k = 0; k < n; k++) {
        System.out.println("Hi");
    }
}

Runs 1 TRILLION times.

Structure matters more than hardware!

Start with a hook. Ask students: If n is 10, both are fast. But if n is 1 million (like user data in an app), Code B will crash your computer. This is why we analyze loops.

1. The Simple Loop


for (int i = 0; i < 5; i++) {
    System.out.println("AP CSA"); // Statement
}

Initialization: Runs 1 time.

Test (i < 5): Runs 6 times (5 true + 1 false).

Update (i++): Runs 5 times.

Body Statement: Runs 5 times.

Let's break it down technically. Usually, in AP CSA informal analysis, we focus mostly on the BODY execution count. How many times does the print statement happen?

Generalizing to n


for (int i = 0; i < n; i++) {
    count++; // How many times?
}

Execution Count: $n$

This is called Linear Time.

If $n$ doubles, the work doubles.

If n is 10, it runs 10 times. If n is 100, it runs 100 times. The relationship is 1:1.

Quick Check

How many times does the statement execute?

for (int i = 0; i < n; i += 2) { ... }

Answer: $\approx n / 2$ times

for (int i = 1; i <= n; i++) { ... }

Answer: $n$ times

Still considered "Linear"!

Interactive moment. Ask the class before showing the answer. Point out that n/2 is still linear because if N doubles, work still doubles (roughly).

2. Nested Loops


for (int r = 0; r < n; r++) {       // Outer Loop
    for (int c = 0; c < n; c++) {   // Inner Loop
        System.out.print("*");      // Statement
    }
    System.out.println();
}

Outer runs $n$ times.

Inner runs $n$ times for every outer iteration.

Total = $n \times n = n^2$

This is Topic 2.11 applied to 2.12 analysis. Visual: Rows and Columns. If n is 10, we print 100 stars. If n is 100, we print 10,000 stars!

The Power of $N^2$ (Quadratic)

N (Input) Linear ($N$) Quadratic ($N^2$)
10 10 100
100 100 10,000
1,000 1,000 1,000,000

Avoid nested loops with large data if possible!

Challenge: Analyze This


int count = 0;
for (int i = 0; i < 5; i++) {
    for (int j = 0; j < 4; j++) {
        count++;
    }
}
// What is the final value of count?

Outer: 0, 1, 2, 3, 4 (5 times)

Inner: 0, 1, 2, 3 (4 times)

Total: $5 \times 4 = 20$

Pause here. Let them calculate. This confirms they understand the "multiplication rule" of nested loops.

3. Dependent Loops (Advanced)


// Notice inner loop depends on i
for (int i = 0; i < n; i++) {
    for (int j = 0; j <= i; j++) {
        System.out.print("*");
    }
}

i=0 (1 run), i=1 (2 runs), i=2 (3 runs)...
$1 + 2 + 3 + ... + n$

Roughly $n^2 / 2$

Still Quadratic!

This is common in AP FRQs (triangle patterns). The math is n(n+1)/2. But informally, we still treat this as similar to n^2 efficiency-wise.

Summary

  • Execution Count: How many times the body runs.
  • Single Loop: usually $n$ times (Linear).
  • Nested Loop: usually $n \times n$ times (Quadratic).
  • Performance: As $n$ grows, $n^2$ gets slow very fast.
