Q1. What is the Time Complexity of Selection Sort?

Answer: O(n2)

Explanation: Selection Sort requires two nested loops. The outer loop iterates n times, and the inner loop iterates to find the minimum element in the remaining unsorted sub-array. This occurs regardless of the initial order of elements.

Q2. Trace: Array [40, 10, 30, 20]. Show the array after the 2nd pass.

Answer: [10, 20, 30, 40]

Step-by-Step:

  • Initial: [40, 10, 30, 20]
  • Pass 1: Identify min (10), swap with index 0. Result: [10, 40, 30, 20]
  • Pass 2: Identify min in remaining [40, 30, 20] which is (20), swap with index 1. Result: [10, 20, 30, 40]
Q3. Selection Sort vs. Bubble Sort: Which performs fewer swaps?

Answer: Selection Sort

Explanation: Selection Sort makes a maximum of n-1 swaps (one swap per pass). Bubble Sort, in the worst case, swaps adjacent elements continuously, potentially performing O(n2) swaps.

Q4. Fill in the blank: The inner loop of Selection Sort starts from _____.

Answer: current_index + 1

Explanation: If the outer loop variable is i (representing the first element of the unsorted sub-array), the inner loop must search for the minimum value starting from i + 1 up to the end of the array.

Q5. True or False: Selection Sort is faster if the array is already sorted.

Answer: False

Explanation: This is a key characteristic of Selection Sort. It performs the same number of comparisons (O(n2)) regardless of whether the array is sorted, reverse-sorted, or random. It does not "know" the array is sorted until it checks every element.
