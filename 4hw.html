Course Review: Sorting Algorithms

1. Key Takeaways (Summary)

Based on our lesson regarding IB Computer Science B2.4.3:

üîç Selection Sort

  • Mechanism: Repeatedly finds the minimum element in the unsorted portion and swaps it with the first unsorted element.
  • Complexity: Always O(n¬≤). It must scan the unsorted list fully every time, even if the array is already sorted.
  • Swaps: Efficient in writing memory. Maximum n-1 swaps total.

ü´ß Bubble Sort

  • Mechanism: Repeatedly compares adjacent elements and swaps them if order is wrong. Largest element "bubbles" to the end.
  • Complexity: Worst case O(n¬≤). Best case O(n) (if optimized with a flag).
  • Swaps: Can perform many swaps (O(n¬≤) in worst case).

Comparison Cheat Sheet

Feature Bubble Sort Selection Sort
Time Complexity (Worst) O(n¬≤) O(n¬≤)
Time Complexity (Best) O(n) (Already sorted) O(n¬≤) (Always same)
Number of Swaps High (Many) Low (Max n-1)
Stability Stable Not Stable

Initial: [45, 12, 30, 8, 20]

  • Pass 1: Find min (8). Swap 8 with index 0 (45).
    Result: [8, 12, 30, 45, 20]
  • Pass 2: Look at [12, 30, 45, 20]. Find min (12). Swap 12 with index 1 (12). (No change visually, but logically processed).
    Result: [8, 12, 30, 45, 20]

May 2019 / SL Paper 1 / Q10

Question 1 (Tracing)

An array contains the following integer elements:

[18, 5, 9, 12, 3]

Apply the selection sort algorithm to sort the array in ascending order. Show the contents of the array after each swap operation.

View Markscheme / Answer

Initial: [18, 5, 9, 12, 3]

  1. Find min (3). Swap with 18.
    Result: [3, 5, 9, 12, 18]
  2. Find min in [5, 9, 12, 18] -> (5). Swap with 5.
    Result: [3, 5, 9, 12, 18] (No change)
  3. Find min in [9, 12, 18] -> (9). Swap with 9.
    Result: [3, 5, 9, 12, 18] (No change)
  4. Find min in [12, 18] -> (12). Swap with 12.
    Result: [3, 5, 9, 12, 18] (No change)

Note: IB marks are usually awarded for showing the correct state after specific passes or the final sorted array with evidence of the method used.

Nov 2017 / SL Paper 1 / Q6

Question 2 (Complexity & Comparison)

(a) State the time complexity of the bubble sort algorithm in the worst case. [1 mark]

(b) Identify one advantage of using selection sort over bubble sort. [1 mark]

View Markscheme / Answer

(a) Answer: O(n¬≤) or Quadratic.

(b) Answer:

  • Selection sort performs fewer swap operations than bubble sort (maximum n-1 swaps).
  • Selection sort is generally faster for writing to memory if write operations are expensive.
